Atributos de inputs

Os exemplos abaixo têm como fonte o W3Schools.



O atributo "value":

Especifica um valor inicial, previamente preenchido.







O atributo "readonly":

Define o campo como sendo de somente leitura. O conteúdo não pode ser modificado, mas pode ser focado e selecionado.

O valor de um campo read-only será enviado normalmente com a submissão do formulário.







O atributo "disabled":

Desabilita um campo. Seu conteúdo não pode ser focado.

O valor de um campo desabilitado não será enviado com a submissão do formulário.







O atributo "size":

Especifica a largura vísivel de um campo, em caracteres. O valor padrão é 20.

Funciona com os seguintes tipos de input: text, search, tel, url, email, password.







O atributo "maxlength":

Especifica o número máximo de caracteres permitidos em um campo.







Os atributos "min" e "max":

Definem valores mínimos e máximos para um campo.

Funcionam com os seguintes tipos de input: number, range, date, datetime-local, month, time, week.








O atributo "multiple":

Especifica que o usuário pode colocar mais de um valor no campo.

Funciona com os seguintes tipos de input: email, file.




O atributo "pattern":

Define uma expressão regular para checar o valor do campo, quando o formulário é submetido.

Uma "expressão regular" é uma sequência de caracteres que define um padrão de busca.

Funciona com os seguintes tipos de input: text, date, search, url, tel, email, password.




O atributo "placeholder":

Define uma "dica" de preenchimento do campo, que o ocupa antes do usuário preencher.

Funciona com os seguintes tipos de input: text, search, url, number, tel, email, password.




O atributo "required":

Determina que um campo é de preenchimento obrigatório, para poder ocorrer a submissão do formulário.




O atributo "step":

Define o intervalo entre valores que podem ser colocados em um campo.

Funciona com os seguintes tipos de input: number, range, date, datetime-local, month, time, week.

É importante saber que esta restrição pode ser contornada usando-se JavaScript, portanto a mesma restrição deve ser checada no lado do servidor.




O atributo "autofocus":

Determina que o campo deve ser automaticamente focado assim que a página carregar.
