Formulários:

São usados para receber dados do usuário.

Possuem atributos importantes: "action" que especifica o form-handler, "method" que designa um método HTTP para o envio dos dados, entre outros.

Funcionam com o auxílio de diferentes elementos, descritos a seguir.

"Inputs":

São os elementos de formulário mais usados, e sintaticamente trata-se de uma tag versátil, cujo tipo é definido por um atributo.

Estes podem ser: textos, radio buttons, checkboxes, botões, etc.

É importante notar que eles precisam do atributo "name" para que seus dados sejam corretamente identificados ao serem enviados.

Uma exceção é o input do tipo "submit", que funciona sem o atributo "name", principalmente se for único.

Este tipo de input pode ter atributo "name", principalmente se houver mais de um, e seja o caso de identificar qual deles foi submetido ao servidor.

Os exemplos, no formulário que pode ser visto ao lado, têm como fonte o W3Schools.

Rótulos:

São usados juntamente com os campos de input em um formulário. Clicar no rótulo leva ao preenchimento do referido input.

São úteis em questão de acessibilidade, com softwares leitores de tela.

O atributo "for" do rótulo deve ser igual ao "id" do input a que se refere.

Seleções:

Elemento usado para que o usuário escolha uma opção, semelhante ao que ocorre com os inputs do tipo "radio", porém aqui é na forma de uma lista "drop-down".

Pode-se agrupar opções semelhantes ou relacionadas, com a tag específica para isso.

Áreas de texto:

Parecem com os inputs do tipo texto, mas aqui costumam ser maiores, e têm sua própria tag com diferentes atributos, para números de linhas e colunas, etc.

Botões:

É recomendável sempre usar a tag "button" com o atributo type também "button". Isto é por razões de como cada browser trata o elemento por padrão.

É interessante perceber como existem diferenças semânticas entre links, inputs do tipo "submit", e botões.

Notadamente, estes possuem um atributo "onclick" que especifica o que devem fazer, o que pode ser o disparo de funções Javascript, por exemplo.

"Fieldsets":

São usados para agrupar CONJUNTOS de CAMPOS que se relacionam de algum modo, em um formulário.

Legendas:

São usadas para descrever um elemento "fieldset", visto acima.

Listas de dados:

Definem listas de opções pré-definidas para elementos de "input".

O atributo "list" do input precisa ser igual ao atributo "id" do datalist.

"Outputs":

Exibem SAÍDA de algum cálculo ou script.
